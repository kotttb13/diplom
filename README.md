# Neuro-Optimizer

Neuro-Optimizer — desktop-система для:
- загрузки нейросетевых моделей,
- оптимизации под edge-устройства,
- валидации качества до/после оптимизации,
- OTA-развертывания на целевые устройства по SSH.

## Быстрый старт (Windows)

1. Установите Python 3.10+.
2. В корне проекта запустите:
   - `setup.bat`
3. Запустите приложение:
   - `.venv\Scripts\activate.bat`
   - `python main.py`

## Что делает `setup.bat`

- создает `.venv`,
- обновляет `pip`,
- устанавливает зависимости из `requirements.txt`.

## Системные требования

- Windows 10/11 x64
- Python 3.10+
- PostgreSQL 14+ (опционально)
- SQLite (доступна как более простой вариант)

## База данных

По умолчанию используется PostgreSQL (см. `app/core/database/initialization_db.py`).

Приложение при старте:
- создает таблицы,
- выполняет schema-compatibility update,
- досеивает lookup-таблицы (`model_format`, `model_type`).

## Работа в UI

1. **Модели**: добавьте исходную модель (`h5`, `keras`, `onnx`, `pb`, `tflite`, `pt`, `pth`).
2. **Устройства**: добавьте устройство и сохраните SSH-параметры (`username`, `password`, `port`) в БД.
3. **Оптимизация**: выберите модель/устройство, профиль и (опционально) данные валидации.
4. **Развертывание**: выберите оптимизированную модель и устройство; параметры подключения подтянутся из БД.
5. **Отчеты**: анализируйте метрики и примененную стратегию оптимизации.

## Docker (опционально, не часть setup)

Docker-лаборатория нужна для тестов SSH/эмуляции устройств и не обязательна для установки приложения.

```bash
docker-compose up -d --build
docker-compose down -v
```

## Структура проекта

- `app/ui/` — интерфейс и пользовательские сценарии
- `app/core/database/` — модели/инициализация/совместимость схемы
- `app/core/repositories/` — слой данных
- `app/core/services/` — бизнес-логика (устройства, оптимизация, валидация)
- `docs/` — эксплуатационная документация
- `setup.bat` — установка окружения на Windows
