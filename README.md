1.	Назначение и запуск приложения
Neuro-Optimizer — это настольное приложение для автоматической оптимизации нейронных сетей под характеристики конкретных периферийных (edge) устройств и их безопасного развертывания. После установки зависимостей и настройки базы данных (см. Приложение Г) запустите приложение, выполнив команду python main.py в корневой директории проекта.
2.	Рабочие инструкции
  Шаг 1. Добавление нейронной модели
    1.	Перейдите на вкладку Модели.
    2.	Нажмите кнопку Добавить модель.
    3.	В открывшемся диалоге укажите:
      o	Название модели: Произвольное имя для идентификации.
       o	Путь к файлу: Нажмите «Обзор» и выберите файл модели (поддерживаются форматы .h5, .keras, .onnx, .pb).
    4.	Тип ("cv") и формат модели определятся автоматически. При необходимости их можно задать вручную.
    5.	Нажмите Добавить. Модель появится в таблице.
  Шаг 2. Добавление и настройка edge-устройства
    1.	Перейдите на вкладку Устройства.
    2.	Нажмите кнопку Добавить устройство.
    3.	Выберите режим подключения:
      o	Автоматическое обнаружение (рекомендуется):
        	Убедитесь, что устройство (например, смартфон Android с Termux) включено, подключено к той же сети Wi-Fi, что и компьютер, и на нем запущен SSH-сервер (sshd).
        	Введите IP-адрес устройства, порт (по умолчанию 8022 для Termux), имя пользователя и пароль SSH.
        	Выберите тип устройства "android".
        	Нажмите Автоматическое обнаружение. Приложение подключится и считает характеристики (архитектура, память, процессор).
      o	Ручной ввод:
        	Выберите тип устройства.
        	Вручную заполните поля: архитектура, свободная память (ГБ), ОЗУ (ГБ), количество ядер CPU.
    4.	Нажмите Добавить устройство. Устройство появится в таблице.
  Шаг 3. Оптимизация модели для устройства
    1.	Перейдите на вкладку Оптимизация.
    2.	В выпадающем списке Выберите модель укажите исходную модель из списка.
    3.	В выпадающем списке Выберите устройство укажите целевое устройство.
    4.	(Опционально, для валидации) Укажите пути к файлам тестовых данных X_test.npy (данные) и y_test.npy (метки).
    5.	Нажмите Запустить оптимизацию.
    6.	Дождитесь завершения процесса. Результат (новая оптимизированная модель в формате .tflite) будет сохранен локально, а в таблице Отчеты оптимизации появятся записи с метриками точности и потерь до и после оптимизации.
  Шаг 4. Развертывание оптимизированной модели на устройство (OTA)
    1.	Перейдите на вкладку Развертывание.
    2.	В выпадающем списке Выберите оптимизированную модель укажите результат предыдущего шага.
    3.	В выпадающем списке Выберите устройство укажите целевое устройство.
    4.	Параметры подключения (IP, логин, пароль) заполнятся автоматически из данных устройства.
    5.	(Опционально) Задайте Секрет для шифрования. Если оставить поле пустым, секрет будет сгенерирован автоматически на основе ID устройства.
    6.	Нажмите Запустить развертывание.
    7.	Дождитесь завершения операции. В таблице Отчеты развертывания появится новая запись со статусом "completed". Оптимизированная модель будет размещена в директории ~/ota_updates/ на целевом устройстве.
3.	Устранение неполадок
  •	Ошибка подключения к устройству: Убедитесь, что устройство и компьютер находятся в одной сети, IP-адрес верен, SSH-сервер на устройстве запущен, а логин и пароль введены правильно.
  •	Ошибка загрузки модели: Проверьте, что файл модели не поврежден и его формат поддерживается приложением.
  •	Долгая оптимизация: Время оптимизации зависит от размера и сложности модели. Для моделей >100 МБ процесс может занять несколько минут.
  •	Ошибка развертывания: Проверьте доступность устройства по сети, наличие свободного места на устройстве и правильность пути к удаленной директории. Подробности ошибки можно найти в логах приложения.

4.	Скрипт начальной настройки окружения Windows (setup.bat)
Данный пакетный файл выполняет базовую подготовку окружения для запуска приложения на операционной системе Windows. Рекомендуется выполнить его перед первым запуском.
          batch
          @echo off
          echo ========================================
          echo   НАСТРОЙКА ОКРУЖЕНИЯ NEURO-OPTIMIZER
          echo ========================================
          echo.
          
          echo 1. Создание виртуального окружения Python...
          python -m venv .venv
          if errorlevel 1 (
              echo Ошибка: Не удалось создать виртуальное окружение.
              echo Убедитесь, что Python 3.9+ установлен и добавлен в PATH.
              pause
              exit /b 1
          )
          
          echo 2. Активация виртуального окружения...
          call .venv\Scripts\activate.bat
          if errorlevel 1 (
              echo Ошибка: Не удалось активировать виртуальное окружение.
              pause
              exit /b 1
          )
          
          echo 3. Обновление менеджера пакетов pip...
          python -m pip install --upgrade pip
          if errorlevel 1 (
              echo Предупреждение: Не удалось обновить pip. Продолжение установки...
          )
          
          echo 4. Установка зависимостей из requirements.txt...
          pip install -r requirements.txt
          if errorlevel 1 (
              echo Ошибка: Не удалось установить зависимости.
              echo Проверьте наличие файла requirements.txt в текущей директории.
              pause
              exit /b 1
          )
          
          echo.
          echo ========================================
          echo   ОКРУЖЕНИЕ УСПЕШНО НАСТРОЕНО!
          echo ========================================
          echo.
          echo Инструкция для запуска:
          echo 1. Активируйте окружение: .venv\Scripts\activate.bat
          echo 2. Настройте подключение к БД (при необходимости)
          echo 3. Запустите приложение: python main.py
          echo.
          Pause

6.	Инструкция по развертыванию на Windows
  •	Системные требования:
    o	Операционная система: Windows 10/11 (64-bit)
    o	Python: версия 3.9 или выше
    o	Дополнительно (опционально): PostgreSQL 14+ для использования в качестве СУБД
  •	Подготовительные действия:
    o	Установите Python 3.9+ с официального сайта python.org. Во время установки отметьте опцию "Add Python to PATH".
    o	(Опционально) Установите и настройте PostgreSQL, если планируется его использование.
  •	Установка приложения:
    o	Скопируйте или клонируйте папку с проектом на локальный диск.
    o	Откройте командную строку (CMD) и перейдите в корневую директорию проекта.
    o	Запустите скрипт настройки окружения: setup.bat
    o	Скрипт автоматически создаст виртуальное окружение и установит все зависимости.
  •	Настройка подключения к базе данных:
    o	Вариант А: Использование PostgreSQL (рекомендуется для production)
      	Откройте файл: app\core\database\initialization_db.py
      	Найдите строки с определением учетных данных:
              python
              username = "postgres"
              userpassword = "1111"
              
      	Измените значения username и userpassword на действительные учетные данные для подключения к вашему экземпляру PostgreSQL.
    o	Вариант Б: Использование SQLite (рекомендуется для тестирования)
      	Откройте файл: app\core\database\initialization_db.py
      	Найдите функцию create_db_engine:
              python
              def create_db_engine(db_url = db_url_postgre):
              return create_engine(db_url, echo=True)
      	Измените значение аргумента по умолчанию на db_url_sqlLite:
              python
              def create_db_engine(db_url = db_url_sqlLite):
              return create_engine(db_url, echo=True)
      
      	При использовании SQLite отдельная настройка учетных данных не требуется.
  •	Запуск приложения:
    o	Активируйте виртуальное окружение (если оно не активировано автоматически):
              cmd
              .venv\Scripts\activate.bat
    o	Запустите основной файл приложения:
              cmd
              python main.py
    o	При первом запуске система автоматически инициализирует базу данных: создаст необходимые таблицы и заполнит справочники начальными данными.
  •	Настройка целевого устройства (Android):
    o	На устройстве Android установите приложение Termux из Google Play.
    o	В Termux выполните команды:
              bash
              pkg update
              pkg install openssh python
              sshd
    o	Узнайте IP-адрес устройства (команда ip addr в Termux или настройки Wi-Fi).
    o	Имя пользователя по умолчанию в Termux имеет вид u0_aXXX, пароль можно задать командой passwd.
  •	Устранение типовых проблем:
    o	Ошибка DLL при импорте TensorFlow
      	Убедитесь, что активировано виртуальное окружение и установлены Visual C++ Redistributable для Visual Studio.
    o	Ошибка подключения к PostgreSQL
      	Проверьте, запущена ли служба PostgreSQL, верность логина/пароля и разрешены ли подключения в файле pg_hba.conf.
    o	Приложение не запускается
      	Проверьте, что все зависимости установлены корректно (pip list) и Python добавлен в PATH.
Важное примечание: Для работы с edge-устройствами убедитесь, что компьютер и устройство находятся в одной локальной сети, и межсетевой экран Windows не блокирует SSH-соединения (порт 8022 для Termux по умолчанию).
